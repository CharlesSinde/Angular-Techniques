This is the Styles example page!

3 high levels of styling placements

  • App.Module or the Global
  • Shared.Module which would be the features you use across majority of the application
  • Feature Module which will bundled components of common features for a given task

You can structure your CSS or SCSS based on where the style fits into your project

For example, if it is a common style that is mandatory across the project then you can place

this in your app.module level css or in the shared.module css.

You can go into the component level css when it is a one of style that you don't want polluting

your project elsewhere.


If you find that a style is now needed higher up the chain then you promote that style upwards.

A component level style can go to feature, then shared and all the way to app level if needed.
